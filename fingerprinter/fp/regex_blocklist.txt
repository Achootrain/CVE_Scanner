# Patterns listed here are pre-loaded into the global _BLACKLIST at

# auto-blacklist in safe_search itself. Pattern strings come from upstream
# nuclei templates / Wappalyzer rules / retire.js definitions, copied

# Format:
#   - One pattern per line.
#   - Lines starting with '#' are comments. Blank lines ignored.
#   - No quoting, no escaping. The line is the literal pattern string as
#     it appeared in the slow-regex log line ("pattern='...'").
#
# How to add a new entry:
#   1. Run a pipeline that hangs.
#   2. In scan_results.stderr find the most recent
#      "[scan +Xs] matchers slow-regex elapsed=X.Xs ... pattern='...'" line.
#   3. Copy the contents between the single quotes into a new line below.
#   4. Re-run; pattern is skipped from then on (calls return None).
